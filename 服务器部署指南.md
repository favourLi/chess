# 自建服务器部署指南（新手向）

本文说明如何把本仓库（3D 中国象棋 + 联机）部署到**你自己购买的 Linux 云服务器**上。按顺序做即可；更偏原理的说明见根目录 `[DEPLOY.md](./DEPLOY.md)`。

---

## 一、你需要提前准备什么


| 项目           | 说明                                                                                                                                                                    |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 一台服务器        | 建议 **Ubuntu 22.04 LTS**（阿里云、腾讯云、AWS 等均可），最低约 **1 核 2G 内存** 可先跑通                                                                                                       |
| 能 SSH 登录     | 用终端（Mac/Linux 自带；Windows 可用 [PowerShell](https://learn.microsoft.com/zh-cn/windows-server/administration/openssh/openssh_install_firstuse) 或 PuTTY）连接 `ssh 用户名@服务器IP` |
| 域名（**强烈推荐**） | 例如 `chess.example.com`，在域名控制台把 **A 记录** 指到服务器公网 IP。没有域名时也可先用 `http://IP` 测试，但浏览器联机、HTTPS 证书会不方便                                                                       |
| 本项目代码        | 在服务器上用 `git clone` 你的仓库，或把压缩包上传后解压                                                                                                                                    |


**说明**：游戏前端是网页，后端是 **Node.js**（Express + Socket.IO）。不能只放静态文件，必须在服务器上**长期运行**一个 Node 进程。

---

## 二、整体流程（心里有个地图）

1. 登录服务器，安装 **Node.js 20**、**Nginx**。
2. 把项目放到某个目录（例如 `/var/www/chess`），执行 `**npm ci`** 和 `**npm run build`**。
3. 配置 **JWT 密钥** 等环境变量，用 `**NODE_ENV=production`** 启动 Node（此时会同时提供网页、`/api`、联机 WebSocket）。
4. 用 **Nginx** 把 80/443 请求**反向代理**到 Node 监听的端口（如 3030），并申请 **HTTPS 证书**。
5. 用 **PM2**（或 systemd）让 Node **崩溃后自动重启**。

下面按步骤写具体命令（默认你是 Linux `bash`，且具备 `sudo` 权限）。

---

## 三、步骤 1：连接服务器并更新系统

```bash
ssh 你的用户名@服务器公网IP
```

进入后执行：

```bash
sudo apt update
sudo apt upgrade -y
```

---

## 四、步骤 2：安装 Node.js 20

推荐使用 NodeSource 一键装 20.x（与 CI 一致）：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v    # 应显示 v20.x
npm -v
```

---

## 五、步骤 3：安装 Nginx（后面用来做 HTTPS 与反代）

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

浏览器访问 `http://服务器IP`，若看到 Nginx 欢迎页即正常。

---

## 六、步骤 4：把项目代码放到服务器

**方式 A：用 Git（推荐）**

```bash
sudo mkdir -p /var/www
sudo chown -R $USER:$USER /var/www
cd /var/www
git clone https://github.com/你的用户名/chess.git chess
cd chess
```

**方式 B：本机打包上传**

在本机项目目录打包（不要包含 `node_modules`），上传到服务器后：

```bash
cd /var/www
unzip chess.zip -d chess
cd chess
```

---

## 七、步骤 5：安装依赖并构建前端

在**项目根目录**（含 `package.json` 的那一层）执行：

```bash
npm ci
npm test          # 可选，确认环境正常
npm run build
```

成功后会出现 `**dist/**` 目录。生产环境下 Node 会在 `**NODE_ENV=production**` 时自动用 `express.static` 托管 `dist/`，浏览器访问同一域名即可打开游戏。

---

## 八、步骤 6：配置环境变量（JWT 密钥必改）

项目**不会自动读取**仓库里的 `.env`，你需要在服务器上单独放一份配置（**不要提交到 Git**）。

### 8.1 生成两个随机密钥

在服务器上执行**两次**，每次复制一段不同的结果：

```bash
openssl rand -base64 48
```

### 8.2 创建配置文件

例如放到 `/etc/chess.env`（需 root）：

```bash
sudo nano /etc/chess.env
```

写入（把下面两行密钥换成你刚生成的**两段不同**字符串）：

```text
NODE_ENV=production
PORT=3030
JWT_ACCESS_SECRET=这里粘贴第一段随机串
JWT_REFRESH_SECRET=这里粘贴第二段随机串
```

若你使用 **HTTPS 且前后端同一域名**（按本文 Nginx 配置即是），一般**不必**设置 `CORS_ORIGIN`。若前端与 API 不同域名，再在文件中增加一行：

```text
CORS_ORIGIN=https://你的前端完整地址
```

保存后限制权限：

```bash
sudo chmod 600 /etc/chess.env
```

---

## 九、步骤 7：用 PM2 启动 Node（推荐新手）

PM2 可在进程退出后自动拉起。

```bash
sudo npm install -g pm2
cd /var/www/chess
```

先载入环境变量再启动（`/etc/chess.env` 里是 `KEY=value` 形式，无 `export` 也可）：

```bash
set -a
source /etc/chess.env
set +a
pm2 start server/index.js --name chess
pm2 save
pm2 startup
```

最后一条会打印一行以 `sudo` 开头的命令，**按提示复制执行一次**，这样服务器重启后 PM2 也会起来。

常用命令：

```bash
pm2 logs chess    # 看日志
pm2 restart chess # 重启（更新代码后常用）
pm2 status
```

确认本机可访问（在服务器上执行）：

```bash
curl -sI http://127.0.0.1:3030 | head -n1
```

应看到 `HTTP/1.1 200` 或 `304` 之类成功响应。

## 十、步骤 8：配置 Nginx 反向代理 + HTTPS

下面假设你的域名是 `**chess.example.com**`，请全部改成自己的域名。

### 10.1 先写 HTTP 反代（证书申请前也能测）

```bash
sudo nano /etc/nginx/sites-available/chess
```

写入（把 `server_name` 改成你的域名；若暂时没有域名，可写成 `_` 或你的 IP，但 Certbot 对纯 IP 不适用）：

```nginx
server {
    listen 80;
    server_name chess.example.com;

    location / {
        proxy_pass http://127.0.0.1:3030;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }
}
```

启用站点并检查配置：

```bash
sudo ln -sf /etc/nginx/sites-available/chess /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

此时用浏览器访问 `http://chess.example.com`（或临时用 `http://服务器IP` 若你 server_name 配的是 IP）应能打开游戏首页。

### 10.2 申请免费 HTTPS 证书（Let’s Encrypt）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d chess.example.com
```

按提示填邮箱、同意协议。成功后 Certbot 会改 Nginx 配置，浏览器用 `**https://chess.example.com**` 访问。

证书会自动续期；可手动检查：

```bash
sudo certbot renew --dry-run
```

## 十一、步骤 9：防火墙（云厂商安全组 + 本机 ufw）

1. **云控制台「安全组」**：放行 **22**（SSH）、**80**（HTTP）、**443**（HTTPS）。
2. 服务器上若启用了 **ufw**：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

**不要**把 Node 的 `3030` 端口直接暴露到公网；只让本机 Nginx 访问 `127.0.0.1:3030` 即可（上文已是这样）。

---

## 十二、步骤 10：验收

1. 浏览器打开 `**https://你的域名`**。
2. **注册 / 登录** 是否正常。
3. 用两台设备或两个浏览器 **联机下一局**，看走子、断线重连是否正常。
4. 若联机立刻失败，打开浏览器开发者工具 → **网络**，看 `socket.io` 是否为 **101 WebSocket** 或轮询成功；同时执行 `pm2 logs chess` 看服务端报错。

## 十三、数据保存在哪里、如何备份


| 路径                        | 内容        |
| ------------------------- | --------- |
| 项目下 `data/db.json`        | 用户、好友、令牌等 |
| 项目下 `data/replays/*.json` | 对局回放      |


建议定期把 `**/var/www/chess/data/`** 整个目录打包备份到别处。升级或迁移时把该目录拷回即可（停服后再拷贝更安全）。

## 十四、以后如何更新版本

在服务器项目目录：

```bash
cd /var/www/chess
git pull
npm ci
npm run build
pm2 restart chess
```

## 十五、常见问题


| 现象              | 可能原因                      | 处理                                                                  |
| --------------- | ------------------------- | ------------------------------------------------------------------- |
| 502 Bad Gateway | Node 没启动或端口不是 3030        | `pm2 status`、`pm2 logs chess`，确认 `PORT` 与 Nginx 里 `proxy_pass` 一致   |
| 页面能开，联机连不上      | Nginx 未转发 WebSocket       | 对照本文 `**Upgrade` / `Connection**` 两行是否齐全                            |
| 提示 JWT / 登录异常   | 密钥改过或环境变量未载入              | 确认 `source /etc/chess.env` 后再 `pm2 restart`，或用 `pm2 show chess` 看环境 |
| `npm ci` 报错     | Node 版本过低                 | 使用 Node **20**                                                      |
| 只有 IP、没有域名      | 无法用 Let’s Encrypt 默认方式签证书 | 先按 HTTP 部署；或购买域名后再上 HTTPS                                           |

## 十六、如果你用的是 Windows Server

思路相同：安装 [Node.js 20 LTS](https://nodejs.org/)，在项目目录执行 `npm ci`、`npm run build`，用**系统环境变量**或 NSSM 等工具把 `NODE_ENV=production` 与 JWT 变量注入到 `node server/index.js`，前面再用 **IIS 反向代理** 或 **Caddy for Windows** 做 HTTPS。细节与 Linux 不同，可单独查阅「Windows Node 部署」教程；联机对 **WebSocket 转发** 的要求与 Nginx 一致。
